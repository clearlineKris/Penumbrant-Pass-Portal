# PPL Maintenance Guide

## Potent Penaural Library (PPL) - Bi-Weekly Maintenance Schedule

This document outlines the maintenance procedures for the Potent Penaural Library (PPL), which provides customized legal insights based on real statute (but NOT legal advice).

## Overview
The PPL consists of two primary collections:
1. **Penumbrant Series** - Papers exploring gray areas in cannabis law
2. **ORA Reports** - Analysis of Operator-Regulator relationships

## Bi-Weekly Maintenance Tasks

### Week 1 & 3 (Odd Weeks)
**Penumbrant Series Updates:**
- [ ] Review federal cannabis policy announcements and DEA schedule updates
- [ ] Check for new state legislation affecting cannabis regulations
- [ ] Update existing state papers with any legal changes
- [ ] Research and identify new states for Penumbrant Series coverage
- [ ] Review and update citations for accuracy

### Week 2 & 4 (Even Weeks)
**ORA Reports Updates:**
- [ ] Monitor state regulatory body announcements and changes
- [ ] Update operator licensing requirements for covered states
- [ ] Document enforcement action trends and regulatory changes
- [ ] Research and identify new states for ORA Report coverage
- [ ] Update contact information for regulatory bodies

### Every Bi-Weekly Cycle
**General Maintenance:**
- [ ] Review README.md links and ensure all references are current
- [ ] Verify document dates and version numbers
- [ ] Check for broken links or outdated references
- [ ] Update the main repository README with new papers/reports
- [ ] Archive outdated versions if significant updates are made

## Sources to Monitor

### Federal Level
- DEA scheduling announcements
- DOJ policy memos
- Congressional bills and hearings on cannabis
- Federal court decisions impacting cannabis law

### State Level
- State legislature websites and bill tracking systems
- State cannabis regulatory body websites
- Attorney General opinions and guidance
- State court decisions

## Document Update Process

1. **Identify Changes**: Note legal or regulatory changes during monitoring
2. **Assess Impact**: Determine which documents need updating
3. **Update Documents**: Revise affected papers/reports with new information
4. **Version Control**: Update document dates and maintain change logs
5. **Update Repository**: Update README.md and commit changes
6. **Review**: Verify all links and references are working

## File Management

### Git-Tracked Files
- README.md files in each folder
- One-pagers and leaflets (summary documents)
- Maintenance documentation

### Git-Ignored Files (per .gitignore)
- Full Penumbrant Papers (50+ pages)
- Full ORA Reports (detailed analysis)

These larger files should be stored externally and linked from the repository.

## Quality Standards

All PPL content must:
- Be based on real legal statute and official sources
- Include proper citations and references
- Avoid providing legal advice (analysis and context only)
- Maintain objectivity and factual accuracy
- Date documents clearly for version tracking
- Use consistent formatting and naming conventions
